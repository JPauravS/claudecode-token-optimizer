### Modifications

- `hooks/caveman-mode-tracker.js` — explicit `off` / `on` / `disable` / `enable`
  handling for the `/caveman` command switch. Upstream fell through to
  `getDefaultMode()` on unknown args, so `/caveman off` wrote `full`. Patch
  applied by `scripts/fetch-caveman.js` (`applyModeTrackerPatch`).
- `skills/caveman-compress/scripts/cli.py` — force UTF-8 on Windows
  `sys.stdout` / `sys.stderr` via `reconfigure()` so emoji in error paths
  (`❌`, `✅`, `⚠️`) don't raise `UnicodeEncodeError` under cp1252.
  Applied at top of entry point. (`applyCliPyPatch`).
- `skills/caveman-compress/scripts/compress.py` — added `encoding="utf-8"`
  to the `claude --print` `subprocess.run` call so Unicode in prompt and
  response round-trips intact. (`applyCompressPyPatch`).

All other vendored caveman files are unmodified.

### Modifications

The patches route OpenWolf's single-`.wolf/` model onto a dual-`.wolf/` layout
(workspace for shared guidance files, project-local for per-project state)
and neutralize upstream features not used here.

- `hooks/shared.ts` — added `getWorkspaceWolfDir()` + `resolveWolfFile()`
  for dual-`.wolf/` routing. See `scripts/patches/shared.ts.patch.js`.
- `hooks/stop.ts` — cerebrum reflection writes routed to workspace
  `.wolf/cerebrum.md`. (`stop.ts.patch.js`)
- `hooks/post-write.ts` — bug detection reads + writes routed to workspace
  `.wolf/buglog.json`. (`post-write.ts.patch.js`)
- `hooks/session-start.ts` — auto-create project `.wolf/` on first session
  when inside configured workspace. (`session-start.ts.patch.js`)
- `hooks/pre-write.ts` — cerebrum + buglog reads routed to workspace.
  (`pre-write.ts.patch.js`)
- `cli/init.ts` — neutralized references to `reframe-frameworks.md` +
  `designqc-report.json` (upstream features not shipped here).
  (`init.ts.patch.js`)
- `cli/bug-cmd.ts` — buglog lookup routed to workspace `.wolf/`.
  (`bug-cmd.ts.patch.js`)
- `cli/status.ts` — dual-`.wolf/` helper routes workspace-owned files.
  (`status.ts.patch.js`)
- `cli/index.ts` — dropped `dashboard` and `designqc` CLI command groups
  (upstream features not shipped here). (`cli-index.ts.patch.js`)
- `daemon/cron-engine.ts` — project-scoped cron state isolation.
  (`cron-engine.ts.patch.js`)
- `buglog/bug-tracker.ts` — `project_origin` provenance + scoped reads.
  (`bug-tracker.ts.patch.js`)
- `templates/OPENWOLF.md` — stripped `## Design QC` and `## Reframe`
  sections (dropped features).
- `templates/config.json` — stripped `designqc` block.
